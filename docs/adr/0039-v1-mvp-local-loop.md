# Define the v1 MVP as a successful local loop

Tasker v1 is validated when a developer can run `tasker init`, create a local-worktree Task Queue, delegate a Root Task, run `tasker work --once`, have pi execute the Worker Agent in a Local Worktree, update structured criteria/validation and the Workpad Note, squash-merge through Agent-Gated Integration into Main Branch, reach Done, clean up local delivery artifacts, and inspect the run through status/metrics. Human Review may be implemented, but it is not required for the MVP happy path.
